# Chapter 2 — Taste Still Matters When Code Gets Cheap

## Role in the book
Prevent the book from becoming a pure automation triumphalism story. Establish that fundamentals, judgment, and craft become more important when generation gets cheaper.

## Strongest claims
1. Cheap code raises the value of taste, architecture, and judgment.
2. AI amplifies both good operators and bad habits.
3. Vibe coding is excellent for exploration but dangerous as a default production method.
4. The new scarce skill is increasingly problem framing and quality discrimination.

## Bridge from Chapter 1
If Chapter 1 argues that the real shift is from suggestion to delegated work, Chapter 2 should answer the immediate human question that follows: what remains scarce when output gets cheap?

The answer is judgment.

Delegation does not remove the need for taste. It makes taste more visible. Once models can generate code, prose, and plans quickly, the bottleneck moves toward problem framing, constraint setting, review quality, and the ability to distinguish elegant output from expensive slop. Chapter 2 should therefore feel less like a cultural detour and more like the human half of the opening argument before the manuscript descends into the technical core.
